Visit Management Demo

createVisit

Creates a visit for the selected attendees and returns the result to a callback. Salesforce applies all configured visit validations. Invoke only once per presentation session.

SYNTAX
PresentationPlayer.createVisit(callbackMethod)

SUCCESS RESPONSE

{"state":"success", "id": "<parent_visit_uid>"}

ERROR RESPONSE

{"state":"error", "errorMessage": "<error>", "id": "<optional_id_of_previously_created_visit>"}
Related records created: Provider visit, Provider visit product detailing, Provider visit detailing product message, Presentation forum, Presentation click stream entry.
Errors occur when: Validation rules fail, a visit was already created this session, the player is opened for an existing visit, or the player is in an unrelated context.

RESULT

No visit created yet.

updateFeedback

Sets the feedback/reaction from the healthcare professional (HCP) to the specified type during presentations. Updates the related provider visit detailing product message record.

SYNTAX
PresentationPlayer.updateFeedback(type) // type: 'Positive', 'Negative', 'Neutral', or null

INLINE USAGE

<a href="javascript:PresentationPlayer.updateFeedback('Positive');"> I like the presentation </a>
Supported values: Positive, Negative, Neutral, and null (clears the feedback).

CURRENT FEEDBACK

No feedback set.
