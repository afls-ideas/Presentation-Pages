createVisit Function

Creates a visit for the selected attendees and returns the result to the specified callback method. When visits are created, Salesforce applies all configured visit validations.

Syntax

PresentationPlayer.createVisit(callbackMethod)

Example

function createVisit() { PresentationPlayer.createVisit('onVisitCreated'); } function onVisitCreated(data) { if (data.state === 'success') { console.log('Visit ID: ' + data.id); } else { console.log('Error: ' + data.errorMessage); } }

Success Response

{ "state": "success", "id": "<parent_visit_uid>" }

Error Response

{ "state": "error", "errorMessage": "<error>", "id": "<optional_previously_created_visit_id>" }

Related Records Created

These records are automatically related to the new visit:

  • ProviderVisit
  • ProviderVisitProdDetailing
  • ProviderVisitDetailingProductMessage
  • PresentationForum
  • PresentationClickStreamEntry
Important: Do not use the upsert function to create visits. Use createVisit instead.
Tip: Invoke this function only once per presentation session. If called again, it returns the ID of the previously created visit.

Limitations

  • Does not populate Mustache variables that reference visits
  • At least one attendee must be selected before calling createVisit (returns error otherwise)
  • Supported only in the Life Sciences Cloud mobile app
