Account Upsert

Create or update Account records. Include id to update; omit to create.

Create Account + HealthcareProvider

On insert, a HealthcareProvider is automatically created and linked to the new Account.

PresentationPlayer.upsert([{ 'sobject': 'Account', 'Name': 'Acme Healthcare', 'Type': 'Customer' }], 'onAccountUpserted'); // In callback, create HealthcareProvider var accountId = data.ids[0]; PresentationPlayer.upsert([{ 'sobject': 'HealthcareProvider', 'AccountId': accountId, 'Name': 'Acme Healthcare' }], 'onHCPUpserted');

Update Account

PresentationPlayer.upsert([{ 'sobject': 'Account', 'id': '001xx000003DGbYAAW', 'Name': 'Acme Healthcare Inc.', 'Type': 'Partner' }], 'onAccountUpserted');

Person Account

All name fields (FirstName, LastName, Salutation, MiddleName, Suffix) and PersonBirthdate are blacklisted for both insert and update. Person Accounts can only be updated for non-name fields like Type.

After upsert: Records are saved to the local database first. Sync to push to the server.

Limits

Max 15 records per call. Cannot create/update User, Record Type, Territory2, or UserTerritory2Association.
