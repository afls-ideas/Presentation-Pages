Alerts & Logging Demo

alert

Opens a native alert dialog on the user's mobile device with the specified message. Useful for confirming actions or displaying important information.

SYNTAX
PresentationPlayer.alert(message)


COMMON USAGE

// After creating a visit function callbackMethod(data) { console.log(data); PresentationPlayer.alert( JSON.stringify(data) ); } // After upserting records function upsertCallback(data) { if (data.state === 'success') { PresentationPlayer.alert('Record saved!'); } else { PresentationPlayer.alert( data.message + '\n' + data.code ); } }
Mobile only. This function opens a native iOS alert dialog. It is not available on desktop.

logError

Logs error messages within the presentation player. Error messages are tracked per individual presentation page. Maximum 10 error messages per page.

SYNTAX
PresentationPlayer.logError(errorMessage)


USAGE IN ERROR HANDLING

function getAccountsCallback(data) { if (data.state === 'success') { // Process records } else { PresentationPlayer.logError( 'Query failed: ' + data.message ); PresentationPlayer.alert( 'An error occurred. Please try again.' ); } }
Limits: Only 10 error messages are logged per presentation page. Errors are tracked and logged for each individual page. In the mobile app, these events appear in the logs.
