Data Query & Upsert Demo

fetchWithParams

Query data not already in Mustache variables. Supports SOQL with SELECT, FROM, WHERE, GROUP BY, ORDER BY, LIMIT.

SYNTAX
PresentationPlayer.fetchWithParams(query, params, callbackMethod)

BASIC QUERY

PresentationPlayer.fetchWithParams( 'SELECT Id, Name FROM Account', {'batchSize': 15}, 'getAccountsCallback' );

PAGINATION WITH QUERYLOCATOR

function getAccountsCallback(data) { // Process data.records if (!data.done) { // Fetch next batch PresentationPlayer.fetchWithParams( data.queryLocator, {'batchSize': 15}, 'getAccountsCallback' ); } }

QUERY WITH MUSTACHE VARIABLES

PresentationPlayer.fetchWithParams( 'SELECT Id, Name FROM Account WHERE Id = \'' + configData.parameters.id + '\'', {'batchSize': 100}, 'callback' );
Limits: Max 100 records per call. Default batch size is 15. Supports User, Account, Record Type, Territory2, UserTerritory2Association, LSC objects, and custom objects.

upsert

Create or update records for Salesforce objects. Requires Salesforce Support to allowlist target objects.

SYNTAX
PresentationPlayer.upsert(objects, callbackMethod)

CREATE RECORD (no id)

PresentationPlayer.upsert([{ 'sobject': 'Account', 'name': 'New Account', 'customField__c': 'value' }], 'upsertCallback');

UPDATE RECORD (with id)

PresentationPlayer.upsert([{ 'sobject': 'Account', 'id': accountId, 'name': 'Updated Name', 'customField__c': 'new value' }], 'upsertCallback');

Limits: Max 15 records per call. Can't create/update User, Record Type, Territory, Territory2, UserTerritory2Association. Can't update related records in one call.
