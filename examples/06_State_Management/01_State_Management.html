State Management Demo

saveState

Saves data about the presentation's state to the JSON state property. Stores the state on the user's mobile device so users can resume where they left off.

SYNTAX
PresentationPlayer.saveState(state)

STATE DATA (JSON)


Storage: State values are stored and saved for each presentation and visit. When users resume a presentation from a visit, they return to their previous state.

READING SAVED STATE

// State is available in the JSON data var configData; document.addEventListener( 'PresentationDOMContentLoaded', function(event) { configData = event.data; var savedState = configData.state; if (savedState) { var state = JSON.parse(savedState); // Resume from saved position } } );

Dismiss Control

Control how the presentation player is closed when users select the Visit button. Use these to execute long-running operations before dismissal.

DISABLE DISMISS
PresentationPlayer.disableDismiss()
ENABLE DISMISS
PresentationPlayer.enableDismiss()
30-Second Rule: If enableDismiss() isn't called within 30 seconds, the presentation player is dismissed automatically.

RECOMMENDED WORKFLOW

// 1. Disable dismiss when page opens PresentationPlayer.registerEventListener( 'viewappearing', function() { PresentationPlayer.disableDismiss(); } ); // 2. On Visit button press, do work then enable PresentationPlayer.registerEventListener( 'returntovisitbuttonpress', function() { // Save data, run upserts, etc. PresentationPlayer.saveState( JSON.stringify({completed: true}) ); // Allow dismissal PresentationPlayer.enableDismiss(); } );
