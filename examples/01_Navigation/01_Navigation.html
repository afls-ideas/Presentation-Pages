Navigation Demo

Page 1 of 3: goNextPage / goPreviousPage

This is a 3-slide mini presentation demonstrating LSC navigation. Each slide is a separate ZIP/HTML file. Use the buttons below to navigate between them.

Page 1 of 3
SYNTAX
PresentationPlayer.goNextPage() PresentationPlayer.goPreviousPage()

Navigates to the next or previous page in the presentation. Does nothing if already on the first/last page. Available on both desktop and mobile.

HOW IT WORKS

// Each presentation page = one ZIP file // The player maintains the page order: // 01_Navigation.zip (this page) // 02_Navigation_PageB.zip (page 2) // 03_Navigation_PageC.zip (page 3) // // goNextPage() moves forward in order // goPreviousPage() moves backward // // Tap "Next Page" above to see it in action!
Desktop + Mobile: goNextPage, goPreviousPage, and goToSlide are the only navigation functions available on desktop. All other PresentationPlayer functions are mobile only.

goToSlide (Jump to Any Page)

Jump directly to any page by Page ID, Page Name, or Source System Identifier. Try jumping to Page 3 from here.

SYNTAX
PresentationPlayer.gotoSlide( identifier, slideName, animation ) // identifier: PageId, PageName, or // SourceSystemIdentifier // slideName: HTML filename inside the ZIP // animation: 'noanimation' | 'swipeleft' // | 'swiperight' | null

JUMP TO PAGE 3

These buttons call goToSlide with different animation options to jump directly to Page 3, skipping Page 2.

GETTING THE PAGE ID

var configData; document.addEventListener( 'PresentationDOMContentLoaded', function(event) { configData = event.data; // Get page IDs from the JSON: var pages = configData.presentations[ configData.presentationIndex ].Pages; // pages[0].id = this page (Page 1) // pages[1].id = Page 2 // pages[2].id = Page 3 } );
Tip: You can also use the Mustache triple-brace shorthand () to get the full JSON inline at render time.
