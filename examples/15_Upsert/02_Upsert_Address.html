Visit Creation Flow

This demo chains three upsert calls to create a complete visit record with product detailing. Each step uses the Id returned by the previous step.

Step 1: Create Visit

PresentationPlayer.upsert([{ 'sobject': 'Visit', 'AccountId': accountId, 'VisitorId': currentUserId, 'PlannedVisitStartTime': '2025-03-15T09:00:00.000Z', 'PlannedVisitEndTime': '2025-03-15T10:00:00.000Z', 'Status': 'Planned', 'TerritoryId': territoryId }], 'onVisitCreated');

Step 2: Create ProviderVisit

Links a healthcare provider to the Visit.

// Use the Visit Id from Step 1 PresentationPlayer.upsert([{ 'sobject': 'ProviderVisit', 'VisitId': visitId }], 'onProviderVisitCreated');

Step 3: Create ProviderVisitProductDetail

Links a product/brand to the ProviderVisit for detailing tracking.

// Use the ProviderVisit Id from Step 2 PresentationPlayer.upsert([{ 'sobject': 'ProviderVisitProductDetail', 'ProviderVisitId': providerVisitId, 'ProductId': brandId }], 'onDetailingCreated');

Data Queries

Before creating the visit, we query for the required reference data:

// Accounts "SELECT Id, Name, Type FROM Account ORDER BY Name LIMIT 50" // Current User (from configData) configData.parameters.id // Territory "SELECT Id, Name FROM Territory2 LIMIT 1" // Healthcare Provider for Account "SELECT Id, Name FROM HealthcareProvider WHERE AccountId = '{accountId}' LIMIT 1" // Brands (Products) "SELECT Id, Name FROM Product2 WHERE IsActive = true ORDER BY Name LIMIT 50"
