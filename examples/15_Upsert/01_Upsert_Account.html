How Upsert Works

Create or update Account records using PresentationPlayer.upsert(). Include an id to update; omit it to create.

Syntax

PresentationPlayer.upsert( objects, // Array (max 15 records) callbackMethod // String function name )

Account vs Person Account

Account (business account) uses the Name field and supports both insert and update.

Person Account uses FirstName and LastName instead of Name. However, all Person Account name fields (FirstName, LastName, Salutation, MiddleName, Suffix) and PersonBirthdate are blacklisted for both insert and update via upsert. Person Accounts can only be updated for non-name fields like Type.

Create Account + HealthcareProvider

When inserting a new Account, a HealthcareProvider record is automatically created and linked to the new Account via AccountId.

// Step 1: Create the Account PresentationPlayer.upsert([{ 'sobject': 'Account', 'Name': 'Acme Healthcare', 'Type': 'Customer' }], 'onAccountUpserted'); // Step 2: In the callback, create // HealthcareProvider with the new Account Id function onAccountUpserted(data) { if (data.state === 'success') { var accountId = data.ids[0]; PresentationPlayer.upsert([{ 'sobject': 'HealthcareProvider', 'AccountId': accountId, 'Name': 'Acme Healthcare' }], 'onHCPUpserted'); } }

Update Account

PresentationPlayer.upsert([{ 'sobject': 'Account', 'id': '001xx000003DGbYAAW', 'Name': 'Acme Healthcare Inc.', 'Type': 'Partner' }], 'onAccountUpserted');

Update Person Account

// Name fields are blacklisted — only // non-name fields like Type can be updated. PresentationPlayer.upsert([{ 'sobject': 'Account', 'id': '001xx000003DGbYAAW', 'Type': 'Medical Doctor' }], 'onAccountUpserted');

Person Accounts cannot be created via upsert. FirstName, LastName, and all other name fields return OCE_UPSERT_FIELD_BLACKLISTED on both insert and update. Even with a valid RecordTypeId, these fields are blocked. Person Accounts can only be updated for non-name fields like Type.
After upsert: Records are saved to the local database first. The user must sync to push the record to the server. Until synced, the record only exists on this device.

Limits

Max 15 records per call. Cannot create/update User, Record Type, Territory2, or UserTerritory2Association objects.
