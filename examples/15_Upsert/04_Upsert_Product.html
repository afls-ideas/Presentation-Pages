Product2 Upsert

Create or update Product2 records. Product2 is the standard Salesforce product catalog object. In Life Sciences, products represent drugs, devices, or therapeutic items that reps discuss during visits.

PRODUCT2 FIELDS
FieldType
sobjectStringREQUIRED
idStringUPDATE ONLY
NameStringREQUIRED
ProductCodeStringOPTIONAL
DescriptionTextOPTIONAL
FamilyPicklistOPTIONAL
IsActiveBooleanOPTIONAL
CREATE PRODUCT
PresentationPlayer.upsert([{ 'sobject': 'Product2', 'Name': 'Immunexis 200mg', 'ProductCode': 'IMX-200', 'Description': 'Immunomodulator for autoimmune disorders', 'Family': 'Immunology', 'IsActive': true }], 'onProductUpserted');
UPDATE PRODUCT
PresentationPlayer.upsert([{ 'sobject': 'Product2', 'id': '01txx0000006iIyAAI', 'Description': 'Updated description with new clinical data', 'IsActive': true }], 'onProductUpserted');
Use case: During a presentation, a rep might update product notes or flag a product status change based on a conversation with a provider.
Note: Product2 is typically managed by admins. Coordinate with your admin before upserting product records from presentations.

Batch Upsert Example

You can upsert up to 15 records in a single call. This is useful for updating multiple products at once.

BATCH CREATE (multiple products)
PresentationPlayer.upsert([ { 'sobject': 'Product2', 'Name': 'Immunexis 100mg', 'ProductCode': 'IMX-100', 'Family': 'Immunology', 'IsActive': true }, { 'sobject': 'Product2', 'Name': 'Immunexis 200mg', 'ProductCode': 'IMX-200', 'Family': 'Immunology', 'IsActive': true }, { 'sobject': 'Product2', 'Name': 'Cardivex 50mg', 'ProductCode': 'CVX-50', 'Family': 'Cardiology', 'IsActive': true } ], 'onProductUpserted');

CALLBACK HANDLER

function onProductUpserted(data) { if (data.state === 'success') { console.log('Product(s) saved!'); // For batch upserts, data contains // results for each record } else { console.log('Error: ' + data.message); console.log('Code: ' + data.code); } }
