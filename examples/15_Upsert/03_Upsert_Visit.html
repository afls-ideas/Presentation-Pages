Visit

Create or update Visit records. Visits represent scheduled interactions between a rep and an account. Typically created by the system, but can be upserted to update status or notes.

Tip: Use the createVisit() function for standard visit creation during a call. Use upsert when you need to update visit details like status, timestamps, or custom fields from within a presentation.

ProviderVisit

Create or update ProviderVisit records. ProviderVisit links a healthcare provider (Account) to a Visit, tracking the provider-specific context of an interaction.

PROVIDERVISIT FIELDS
FieldType
sobjectStringREQUIRED
idStringUPDATE ONLY
AccountIdReferenceREQUIRED
VisitIdReferenceREQUIRED
StatusPicklistOPTIONAL
NotesTextOPTIONAL
CREATE PROVIDERVISIT
PresentationPlayer.upsert([{ 'sobject': 'ProviderVisit', 'AccountId': '001xx000003DGbYAAW', 'VisitId': '0Bqxx0000008OQHCA2', 'Status': 'Planned', 'Notes': 'Discuss new formulary' }], 'onProviderVisitUpserted');
UPDATE PROVIDERVISIT
PresentationPlayer.upsert([{ 'sobject': 'ProviderVisit', 'id': '0pvxx0000004ABCDEF', 'Status': 'Completed', 'Notes': 'Discussed formulary changes. Provider agreed to trial period.' }], 'onProviderVisitUpserted');
Visit vs ProviderVisit: A Visit is the scheduled event. A ProviderVisit links a specific provider to that visit. One Visit can have multiple ProviderVisits if multiple providers are seen during the same appointment.
